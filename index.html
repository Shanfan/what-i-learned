HTML Basics

How the "Web" is structured?

HTML is the basic document format that contains all the contents the author wants to share with the rest of the web.

Many HTML files are hyperlinked through URLs. By hyperlinking the URLs, we'll be able to navigate from one file to another, hence the "web".

HTTP is the internet protocol that allows my browser to access the files stored on the server.

HTML Basics

HTML files are written with tags and attributes.

The open tag and its close tag together with their enclosed content are called Element. Note that some tags are void tag which by itself is an element. Such as <br> or <img>. Some tages are inline tag, which stays in the line of the text. Some others are block tag, which create a rectangle box surrounding the text or image and block the content off the flowing line of the previous text. There's also another type of tag called Container Tags. Unlike 'paragraph element' or 'header 1 element' which specifies the importance or purpose of the text, the container tag is invisible, and doesn't serve any specific content but to just contain the elements.

Why Use CSS

There're 3 ways to tell HTML what each of its elements should look like:

  1. Inside the element, define the style as element attribute:
    <h3 style="color: #555; font-size: 18px; font-style: italic;">Title</h3>
  2. Inside the <head> of the HTML file:
    <html>
	<head>
		<title>My Page</title>
		<style>
			h3 {
				color: #555;
				font-size: 18px;
				font-style: italic;
			}
		</style>
	</head>
	<body>
	...
  3. Write the CSS in a separate file, and link to that file from the HTML:
    <html>
	<head>
		<title>My Page</title>
		<link rel="stylesheet" type="text/css" href="style.css">
	</head>
	<body>
	...

The first method is not recommended. The last method is recommented.

When we write code, one basic principle is DRY, which stands for Don't Repeat Yourself. When you put all the style definition in each and every HTML element, understandably you'll ended up writing a lot of repeatitive code, like defining what the paragraph font should be, again and again, every time you have a new paragraph element. The second method is better than the first, because it allows you to define, for instance the <p> once, then you'll be able to apply that same style to all the <p> appeared in this HTML file.

However, the last method is the most recommended, especially when we're dealing with large websites where there're tons of HTML files that need to follow the same graphic style. By writing a separate CSS file, we can simply link to that same CSS from multiple HTML files. This way we avoided repeating ourselves.

The "Don't Repeat Yourself" principle is not just so that we can write less code; It's also important when we're debugging or dealing with changes. If the designer decided to change the color of all the hyperlinks in this website... I would be thanking myself for writing a separate CSS file.

Cascading Style Sheets

"Cascading" means that rules are applied not only to the elements they directly match, but also to all of those elements' child elements. However, if a child element has multiple, overlapping rules defined for it, the more specific rule takes effect.

"Serious" Programming

The difference between a toaster and a computer

A toaster has designated function - to toast. Even though you can add advanced features such as setting a timer, automatically adjusting the temperature based on thickness of the bread... etc., its function is still to toast.

A computer on the other hand, is an universal machine that can be programmed to perform different functions. A computer without any program installed on it is a dumb machine that can't do anything. It doesn't have any designated function. But programs do. Each and every program has a disgnate function to fulfill. Once a computer is programmed, it can do anything - any computational thing.

So what is a program? A program is a specifically and strictly defined sequence of steps. When we program, we're essentially designing the sequence of steps the computer needs to take to fulfill the function we want. The program communicates our ideas to the computer in the language a computer understands.

Natural Language vs. programming language

Natural language like English or Mandarin, is ambiguous and verbose. For instance the word 'biweekly' can mean either twice a week or every two weeks. Natual language relies a lot on the context to express its precise meaning, hence its verbosity.

Programming language follows a much more strict grammar, which doesn't leave any room for syntax error or subjective interpretation.

String Indexing in Python

Understand s[x] and s[x:y]:

  • The stirng s is considered as an array of characters.
  • In the first instance, x is the index position of that character. x belongs to range [0, s.length). If x goes beyond s.length-1, s[x] is undefined.
  • In the second instance, x and y can be any integer. If both x and y are within the string's range, and if x is smaller than y, s[x:y] outputs the characters between and include s[x] and s[y-1].
  • If x equals or is larger than y, or if x and y are out of the range of the string, then s[x:y] will output an empty string. It's still a valid expression.

The caveat here is, when you refer to a index range s[x:y] it will always be a valid expression, even if x or y is referring to a position that doesn't exist in the string.

But when you refer to a position, s[x], x has to be an index position s does have. For instance,

s = ''
print s[0]    # Error

This would throw an error because s is empty, thus s[0] doesn't exist.

Also pay attention to s[x] when x < 0. s[-1] is referring to the last letter in the string, when the string is not empty.

Procedure (a.k.a Function)

What Dave kept referring to as a procedure, is more commonly known as a function in programming. A function, is a strictly defined steps that tells the computer how to compute until it reaches to a result.

We can read a pseudocode function below to understand what it means to design a function with a sequence of steps:

def french_toast(a_piece_of_bread):
	if a_piece_of_bread is soft and fluffy: # make sure the bread is not toasted yet
		turn_on_heat(300_degree)
		set_timer(30)
		flip(a_piece_of_bread)
		set_timer(20)
		trun_off_heat()
	return a_piece_of_bread

In the French Toast example above, the function takes a piece of bread that is soft and fluffy as an input, and outputs a piece of bread that is toasted. Anything happens between the statement def french_toast(a_piece_of_bread): and return a_piece_of_bread is the defined procedure.

Input & Output

Input is what you feed to a procedure (a soft and fluffy piece of bread). Output is the result (a toasted piece of bread) of the procedure (toasting).

When we define a function to produce some result, we need to write an expression starting with return. Otherwise the function won't give you any output.

Mutation

Mutation is a very important concept. So pay attention and practice a lot until you internalize this.

String cannot mutate; a list can mutate. What this means? In the explanation below, s is a string, and p is a list.

When you change one letter in the string, the entire string changes; because a string is considered as one single variable. So this is wrong:

s = "string"
s[0] = "r"
print s    
#>>> Error: 'str' object does not support item assignment

When you change one letter in the list, you just change the letter stored in that position, not the other letter or the entire list. In other words, a list is a collection of many variables. So this is correct:

p = ['s', 't', 'r', 'i', 'n', 'g']
p[0] = 'r'
print p   
#>>> rtring

The example below shows what result the mutation of an object can be:


p = ['s', 't', 'r', 'i', 'n', 'g']
q = p
p[0] = 'Y'
print q      
#>>> ['Y', 't', 'r', 'i', 'n', 'g']

When we write down p = ['s', 't', 'r', 'i', 'n', 'g'], an object ['s', 't', 'r', 'i', 'n', 'g'] is created, and it exists with or without p. After this expression, we just assigned a name p to the object ['s', 't', 'r', 'i', 'n', 'g'].

When we write q = p, we instructed the computer to create a new variable name called q, and pointed q to the same object ['s', 't', 'r', 'i', 'n', 'g'].

When we changed the value of p[0] from 'r' to 'Y', we were in fact changing the object ['s', 't', 'r', 'i', 'n', 'g'] to ['Y', 't', 'r', 'i', 'n', 'g']. since q is referring to the same object as p, so q is changed too, without any explicit assignment.

Aliasing

Aliasing: when we have two different ways to refer to the same object, it's called aliasing. Think "James Bond" and "007".

When you say:

p = ['J', 'a', 'm', 'e', 's']
q = p

you created an alias q, which refers to the same object that p is pointed to. So if you change anything with p[i], q is changed too. But if instead of q = p, you wrote q = ['0', '0', '7'], whatever you do to p won't have any effect on q. Because q is pointing to a different object ['0', '0', '7'].

What does it mean by "Object can be mutated"?

Compare these two functions:

spy = [0,0,7]
def replace_spy(p):
    p[2] += 1
    #return p

replace_spy(spy)
print spy
#>>> [0,0,8]
# The object spy is mutated.

spy = 3
def replace_spy(p):
    p += 1
    #return p

replace_spy(spy)
print spy
#>>> 3
# The value of spy is not affected by the replace_spy function

So an object like [0,0,7] can be mutated by a function that only does assignment without a return statement. A variable cannot be mutate in the same way. If you want to change the value of a variable, you have to re-point it to a different value, hence the return mandate.
