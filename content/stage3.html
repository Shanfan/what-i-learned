OOP & Abstraction

Object Oriented Programming (OOP), is a programming paradigm based on the idea of Abstraction.

Abstraction is the idea that you don't need to know everything behind the curtain, while being able to make use of the code written by others.

Because the complexity of the system can be managed by abstraction, we as programmers can be freed from lower-level repetitive works. For example, the CSS classes we learned to write is an example of abstraction.

Modules serve as a good example of abstraction in ways Python works. You don't need to know how the time module is constructed in Python, you can already use that module, such as calling time.current() to display the current time.

Abstraction is a relative concept. In Python's vocabulary, a module is an abstraction of the statements and definitions you want to reuse. The modules can also be "abstracted" into a package, which organizes a collection of modules interreferencing each other.

How Abstraction Works

In this course, we have used webbrowser module, os module, time module and turtle module, all from the Python core modules. To demonstrate the power of abstration, we did an exercise to create a program that checks the profanity words from a text file.

Constructing Your Own Class

Class Constructor

Besides the pre-defined classes by Python, we can also write our own classes.

A class definition starts with a Constructor, which is a function named __init__, and it takes in some attributes of the the object as its arguments.

class Cat():
	'''Kitty Cat'''
	sound = ["meow", "miau", "purr"]
	def __init__(self, hair_type, hair_color, eye_color):
		self.hair_type = hair_type
		self.hair_color = hair_color
		self.eye_color = eye_color

	def happy(self):
		return Cat.sound[2]

lover = Cat("short hair", "calico", "brown")

print "Lover is a {color} {hair} {definition}.".format(color = lover.hair_color, hair=lover.hair_type, definition=lover.__doc__)
# lover.hair_color is an object attribute.

print "When she's happy, she would {}.".format(lover.happy())
# lover.happy() is an object method.

# >>> Lover is a calico short hair Kitty Cat.
# >>> When she's happy, she would purr.

In there above example, notice there're two types of veriables, the Class Variable and the Instance Variable. Instance variables are specific to each instances, such as long hair or short hair, blue eyes or brown eyes... etc. Class varibles are universal to all cats: all of the cats can make these sound = ["meow", "miau", "purr"].

Instance variable: A variable that is defined inside a method and belongs only to the current instance of a class.

Class variable: A variable that is shared by all instances of a class. Class variables are defined within a class but outside any of the class's methods. Class variables aren't used as frequently as instance variables are.

Narratives, Tutorials & References

			

				I started to follow Brian Will on YouTube lately. He gives very succinct lectures to explain the entire thing about programming, computer science... etc. In one of his intro videos, he says, on your journey of learning to program, you're going to encounter 3 types learning materials:
			

			
    
				
  • Narratives
    
					The goal of a narrative is to help you build a conceptual model in your head about the subject matter, so that when you come across a problem, you know how to approach it, where to investigate.
				
    • 
				
  • Tutorials
    
					A tutorial is a demonstration of a way to solve one particular problem. It's always good to attempt a problem yourself at first, then to watch the tutorial to compare your solution with others'.
				
    • 
				
  • References
    
					You consult a reference of the subject matter when you don't remember a certain concept or rule. But if you haven't built a conceptual model in your head, the reference is hardly useful.
				
    • 
			

			

				 It's important to know the distinctions. Having the awareness of what kind of learning material we were studying at that moment, will help us better assess our knowledge and skills. 
			
 
			

				If you have done a lot of tutorial exercises, and you have the access to a good reference, you may have the illusion of knowing it all. However, without a strong conceptual model in your mind, you won't be able to find innovative usage of a technology. This is a problem I noticed in many online courses I've taken.
			

			

				There's nothing wrong to learn from tutorials and references. Sometimes we'll even be able to form a narrative of our own after having done lots and lots of exercises. We just need to be aware.
